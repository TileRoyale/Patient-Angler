01 Treasure Hunter WARN
Code: PASS. Weighted event selection is correctly implemented. lost_treasure weight = 1 + (level × 0.05); all other events weight = 1. At L10, Lost Treasure is 1.5× more likely to be selected than any individual non-treasure event. The bonus applies at runtime during every triggerSpecialEvent() call.
UI description mismatch. Shop text reads "+5% Lost Treasure event chance per level." This implies +5 percentage-point probability gain per level (+50 pp at L10 → 70% chance), which is impossible. The code adds 0.05 to the event's weight, not to its probability directly. With a 5-event pool at L10, the actual probability is 27.3% (up from 20.0% baseline) — a +7.3 pp / +36% relative gain. The upgrade works and benefits the player, but the copy is misleading. No gameplay bug.
